# CafeConnect — промт для применения редизайна (Flutter / main.dart)

Этот документ описывает **все изменения дизайна и логики**, которые нужно внести в существующий проект CafeConnect, чтобы привести его к новой версии (см. интерактивный макет `CafeConnect Staff.dc.html`). Приложение становится **только для персонала**: клиентская часть и QR убираются.

Ты — ведущий Flutter-разработчик. Сохраняй архитектурную целостность (Provider `CafeState`, GoRouter, Hive, `MockCafeApi`) и переиспользуемые компоненты (`AppButton`, `AppCard`, `CategoryChip`, `StatusBadge`). Меняй дизайн-токены централизованно.

---

## 0. Главные принципы редизайна

1. **Только персонал.** Удалить роль «Клиент», экраны заказа клиентом, корзину, статус-трекер клиента и **всю QR-систему** (она уже была удалена — добить остатки).
2. **Автологин в Столы.** Стартовый экран — **сетка столов** (не админка, не логин). Экран логина убрать из дефолтного маршрута (оставить как опциональный для смены пользователя).
3. **Тёплая палитра без конфликтов.** Главная жалоба — синяя кнопка спорит с синей зоной «Бар» и встречается белый текст на белом фоне. Решение: **эспрессо/графит** для всех кнопок и активной навигации; **оранжевый = Кухня**, **синий = Бар** используются СТРОГО как цвета зон (точки, бейджи, полоски карточек заказов, табы), а не для обычных кнопок.
4. **Мягкость и гладкость.** Мягкие тени (вместо «теней нет»), скругления 16–22px, плавные нажатия (scale 0.97 + spring), тактильный отклик.
5. **Скорость и понятность.** Минимум слов, крупные тач-зоны (≥44px), один цвет = один смысл.

## 2. Навигация (нижний таб-бар = весь персонал)

5 вкладок: **Столы · Заказы · Меню · Чаты · Панель**.
- Активная вкладка — иконка/подпись цветом `ink` (bold). Неактивная — тёплый серый `#A8A091`.
- **Бейджи:** на «Заказы» — красный счётчик заказов с задержкой (>20 мин); на «Чаты» — оранжевый счётчик непрочитанных. Бейдж = squircle 18px, белый текст, обводка цветом фона таб-бара.
- Таб-бар: фон `rgba(252,250,245,.92)` + `BackdropFilter(blur 22)`, верхняя линия `line`.
- Старые роли-маршруты в GoRouter свести к этим 5 разделам + оверлеям (детали стола, чек, пересылка, карточка блюда). Удалить маршруты клиента/QR/корзины/статуса.

## 3. Экран «Столы» (home)

Заменяет старую сетку. Что меняется:
- **Шапка:** «Столы», подзаголовок «Зал 1 · N активных · M свободно». Справа — мягкая кнопка-иконка фильтра в белом squircle с тенью.
- **Поиск** (новое): строка поиска по номеру стола и официанту, белая карточка с мягкой тенью.
- **Легенда-фильтр:** горизонтальные пилюли со статусами (Все/Свободен/Новый/Занят/Готов/Задержка) с цветной точкой; активная — фон `cta`, текст белый. Тап фильтрует сетку.
- **Сетка карточек столов** (3 колонки, опц. 4 для плотного зала):
  - Мягкая белая карточка, скругление 20, тонкая тёплая рамка `#F0EBE1`, тень `shadowCard`.
  - Слева сверху — `#N` (мелко, ink3). Справа сверху — статус-точка с «гало» (мягкое кольцо цветом-подложкой), пульс для «Новый/Готов», мигание для «Задержка».
  - По центру — крупный номер стола (Bold), статус-пилюля (фон-подложка + цвет статуса), сумма заказа (или «свободен»).
  - Карточка «Задержка» — рамка цвета `danger` с мягким мерцанием.
- **Жесты (ключевое):**
  - **Тап (onTap)** → открыть **детали стола** (редактирование + заметки).
  - **Долгое нажатие (onLongPress)** → **быстрый «чек» стола** во весь экран, как Reels в Instagram (см. §4). Реализация: `GestureDetector(onTap:…, onLongPress:…)` + `HapticFeedback.mediumImpact()` на long-press. Карточка слегка «придавливается» (scale 0.97) во время удержания.
- Подсказка-коачмарк под сеткой: «Нажми — открыть стол · Зажми — быстрый чек» (можно скрывать после первого раза, флаг в Hive).

## 4. Оверлей «Чек стола» (Reels-стиль) — по долгому нажатию

Полупрозрачный тёмный фон (`rgba(13,11,8,.55)` + blur 14). По центру — карточка-чек, которая **появляется с «pop»-анимацией** (scale 0.86→1 + лёгкий подъём, spring), как превью Reels.
- Сверху тонкая цветная полоса зоны (оранж/синяя по составу заказа).
- Заголовок «Стол NN», подзаголовок «Открыт 14:05 · 3 гостя · Елена», статус-пилюля.
- Тело — «чековые» строки моноширинным: `2×  Флэт уайт ............ 9.00 $`, разделители пунктиром, «Итого» крупно моно.
- Низ: кнопки «Переслать» (вторичная, фон `bg`) и «Открыть стол» (первичная `cta`).
- Подпись «Отпустите, чтобы закрыть». Закрытие — по тапу на фон / отпусканию.
- В коде: показывать как `showGeneralDialog` с `transitionBuilder` (scale+fade), barrierColor затемнение. Это «лупа», не bottom-sheet.

## 5. Экран «Детали стола» (по тапу) — редактирование + заметки + пересылка

Полноэкранный под-экран (back-стрелка):
- Шапка: «Стол NN», «Открыт 14:05 · Елена», статус-пилюля.
- **Карточка текущего заказа** (чек): строки с кол-вом/названием/суммой моно, «Итого». Если пусто — «Стол свободен».
- **Заметки к столу** (новое, ключевое): чипы существующих заметок (тёплый янтарный фон `#FBF3E6`, текст `#A86A24`, иконка-флажок). Поле «Новая заметка: аллергия, ДР, VIP…» + кнопка «+». Заметки хранить в модели стола (список строк), персистить в Hive. Это то, что просил пользователь — «делать пометки к столу».
- **Смена статуса:** ряд пилюль (Свободен/Занят/Новый/Готов/Задержка), выбор подсвечивается цветом статуса. Меняет `table.status` в `CafeState`.
- (Опц.) степпер гостей, переназначение официанта.
- **Действия:** «Переслать» (открыть лист пересылки §6) и «Показать чек» (открыть Reels-оверлей §4).

## 6. Пересылка стола в чат (как в Telegram) — НОВОЕ

Bottom-sheet «Переслать» (вызывается из деталей стола или из чека):
- Ручка-«грабер» сверху, заголовок «Переслать», крестик.
- **Превью пересылаемого** — карточка с левой цветной полосой (как форвард в Telegram): «Стол NN · счёт», краткий состав, «Итого». Подпись «Карточка стола откроется по тапу».
- Поле «Добавить комментарий…».
- Раздел «КУДА ОТПРАВИТЬ» — список целей: **Кухня, Бар, Общий чат** + конкретные сотрудники (аватар squircle, имя, подзаголовок «N онлайн»/роль, круглая кнопка-самолётик).
- По тапу на цель — в выбранный чат добавляется **сообщение-карточка пересланного стола** и происходит переход в этот чат.

### Сообщение-карточка пересланного стола (в диалоге)
- Белая карточка с тенью, шапка «↗ Переслано · Елена · время», левая цветная полоса зоны.
- «Стол NN · счёт», чековые строки моно, «Итого».
- Кнопка-стрелка «Открыть стол» → открывает детали этого стола (§5). Это «интерактивная карточка-ссылка», как уже было для заказов, но теперь и для столов.

Модель: тип сообщения `MessageKind { text, tableCard, orderCard }`; для `tableCard` хранить `tableId` (+ снимок состава на момент пересылки, чтобы чек не «уплывал»).

## 7. Экран «Заказы» (Кухня / Бар) — полировка

Оставляем логику (нравится), обновляем вид:
- **Сегмент-контрол зон** вместо свайп-табов сверху: подложка `surfaceSunken`, активный сегмент — белая «таблетка» с лёгкой тенью; точка-цвет зоны + счётчик. Свайп между зонами оставить как доп.жест.
- **Карточка заказа:** скругление 18, мягкая тень, **верхняя полоса 4px цвета зоны** (оранж/синий). Шапка: squircle-бейдж с номером стола в цвете зоны, «Стол NN», «#id · Кухня/Бар», **моно-таймер** справа (зелёный <15м, оранж 15–20м, красный >20м; у просроченных — мягко мерцающая рамка `danger`).
- Список позиций: `кол-во×` цветом зоны + название.
- Действия: «Отметить готовым» (первичная `cta`, после нажатия → зелёная «Готово ✓») + квадратная кнопка «обсудить в чате» (иконка чата) — открывает соответствующий чат (Кухня/Бар).

## 8. Экран «Меню» (витрина для персонала) — НОВАЯ вкладка

Из-за объединения админки/менеджера нужна **отдельная вкладка-витрина** только для просмотра (официант быстро смотрит и рассказывает гостю). НЕ редактирование (оно в Панели §9).
- Шапка «Меню», подзаголовок «Витрина для персонала · состав, аллергены, время». Поиск.
- Чипы категорий (активный — `cta`).
- Сетка 2 колонки. Карточка: фото 1:1 (скругл. 13), на фото — **бейдж времени приготовления** (часы + «4м») и **точка зоны** (оранж/синий squircle). Под фото — название, цена, **индикатор наличия** (зелёная точка «В наличии» / красная «Стоп-лист»). Для стоп-листа — затемнение фото + плашка «Стоп-лист».
- Тап → **карточка блюда (bottom-sheet, справочная):** фото 16:10, бейдж зоны + время, название + цена, статус наличия, блок **СОСТАВ** (текст), блок **АЛЛЕРГЕНЫ** (чипы; если нет — зелёный чип «Без аллергенов»). Кнопка «Готово». Без «добавить в корзину» — это справочник.

## 9. «Панель» — объединённые Админ + Менеджер (НОВОЕ объединение)

Главная просьба: **объединить метрики и админ-панель**, чтобы функционал был в одном месте и смотрелся гармонично. Решение — один экран «Панель» с верхним сегмент-меню из 4 вкладок: **Обзор · Команда · Меню · Доступ**.

### 9.1. Обзор (метрики — объединены)
- Сетка 2×2 метрик: **Выручка сегодня**, **Средний чек**, **Активные столы (N/12)**, **Ср. время готовки** (+ «N с задержкой» красным). У каждой — цветная точка-категория и дельта (▲/▼) цветом ok/danger.
- Карточка **«Выручка по часам»** — столбчатая диаграмма (бары `#E4D7C2`, пиковый — `cta`), подписи часов. (В вёрстке используйте явные пиксельные высоты баров, не проценты, чтобы не «схлопывались».)
- При желании — мини-блок «Очередь/загрузка зала».

### 9.2. Команда (бывшая админ-CRUD)
- Фильтр-пилюли по ролям (Все/Официанты/Кухня/Бар/Менеджеры).
- Список сотрудников: squircle-аватар с инициалами в цвете роли, имя, «роль · статус», индикатор онлайн (зелёная точка) / оффлайн (серая). Справа — кнопка-карандаш (редактировать). Свайп влево — редактировать/деактивировать.
- Кнопка «Добавить сотрудника» (первичная `cta`). CRUD-форма: имя, роль, права, активность.

### 9.3. Меню (управление — с простым текстом)
- Список позиций: миниатюра, название, цена, бейдж зоны, **переключатель наличия** (зелёный = в наличии / серый = стоп-лист) — мгновенно меняет витрину §8 и стоп-лист на кухне.
- Кнопка «Добавить позицию». **Форма приёма свободного текста:** поля «Название», «Состав» (многострочный текст), «Цена», выбор зоны (Кухня/Бар), наличие. **Фото необязательно** — можно просто оставить текст (как просил пользователь: «при добавлении/изменении меню можно просто оставить текст»). Цена редактируется инлайн.

### 9.4. Доступ (права ролей — админ)
- Карточки ролей (Официант / Повар-Бармен / Менеджер / Администратор) с цветной точкой и числом людей. Внутри — чипы прав: включённые зелёные (с точкой ok), выключенные серые. Тап по чипу — переключение права. Это бывшая «управление правами» из админки, в едином стиле с менеджером.

Старые отдельные экраны «Админ» и «Менеджер» удалить/смержить в этот экран. Доступ к вкладкам «Команда/Меню/Доступ» — по правам (официанту видна только «Обзор» или ничего).

## 12. Тактильный и звуковой отклик (важно для общепита)

Использовать `HapticFeedback` + опц. короткие звуки (`SystemSound`/`audioplayers`):
- Long-press по столу (открытие чека) → `mediumImpact`.
- Смена статуса стола / «Отметить готовым» → `selectionClick` + успех `lightImpact`; новый заказ на кухню → `mediumImpact` + мягкий «пинг».
- Просрочка >20 мин → разовый `heavyImpact` + предупреждающий звук.
- Отправка сообщения / пересылка → `selectionClick`.
- Стоп-лист вкл/выкл → `selectionClick`.
- Все звуки — тихие, отключаемые в настройках (флаг в Hive `soundEnabled`).

## 13. Анимации / нажатия

- Кнопки и карточки: `scale 0.97` на нажатии, возврат spring (`flutter_animate` или `AnimatedScale`).
- Появление списков: fade + `translateY(10→0)`, stagger 40–50ms.
