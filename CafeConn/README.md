# CafeConnect — живая карта зала в кармане у каждого официанта

## Что это

CafeConnect — внутреннее приложение для персонала кафе и ресторана. Не касса — касса остаётся в
r_keeper, iiko или другой системе. CafeConnect координирует зал, кухню и бар: официанты видят
статус каждого стола в реальном времени, принимают заказ прямо с экрана и отправляют его
на нужную станцию (кухня / бар). Заказ сохраняется локально и переживает перезапуск приложения.

## Статус

`v0.1.0-alpha` — первая публичная альфа. Локальное хранилище, один тенант, без realtime-бэкенда
между устройствами. Весь основной поток официанта работает; realtime-синхронизация — это слой 2.

## Что работает в альфе

- **Карта зала** — сетка столов с цветовыми статусами, количеством гостей, именем официанта.
- **Детализация стола** — открытие стола, заметки, текущий заказ, история статусов.
- **Закрытие стола** — с двойным подтверждением (dialog перед `closeTable`).
- **Приём заказа** — iOS-Photos-style multi-select на сетке меню, qty-степпер на каждой карточке.
- **Precheck-лист** — перед отправкой: позиции с редактируемым количеством, заметка к блюду
  (пресеты + свободный ввод), выбор стола чипами (если открыт из таба Меню), превью сплита
  кухня/бар, итог, одна кнопка «ОТПРАВИТЬ ЗАКАЗ».
- **Авто-сплит** — еда идёт на кухню, напитки/кофе — в бар, автоматически.
- **Лонг-пресс на столе → быстрый чек** (Reels-оверлей) — 380 мс удержание, без меню.
- **Лента заказов** — КУХНЯ/БАР по тапу (не свайпу), таймер цветной (<15м зелёный, 15–20м
  оранжевый, >20м красный), кнопка «Готово».
- **Командные чаты** — список чатов, тред, receipt-карты для заказов кухни/бара.
- **Панель управления** — обзор выручки, команда, управление меню, права ролей.
- **Hive-персистентность** — столы, заказы, заметки, настройки живут после перезапуска.
- **Inter + JetBrains Mono** — шрифты в бандле, работают офлайн без google_fonts.

## Чего ещё нет (ограничения альфы)

- Нет realtime-синхронизации между устройствами — каждое устройство живёт само по себе.
- Нет бэкенда, нет API, нет облака.
- Нет интеграции с кассой — оплата проходит отдельно.
- Нет гостевого web-канала (QR-меню для гостей) — это отдельный продукт.
- iOS-edge-swipe-back для pushed-роутов не работает — нужен рефакторинг под parent navigator.
- Нет фискализации и чековой печати — не планируется (это задача кассы).

## Stack

Flutter 3.x · Dart 3 · Provider · GoRouter · Hive · flutter_animate · Inter · JetBrains Mono

## Запуск локально

```bash
flutter pub get
flutter run
```

Минимальная версия Flutter: 3.x. Целевая платформа — Android (APK). iOS не тестировалась.

## Сборка APK

```bash
flutter build apk --release --split-per-abi
```

APK будет в `build/app/outputs/flutter-apk/`. Для теста шрифтов установите с **выключенным
интернетом** — это гарантирует, что шрифты не подтягиваются из сети.
